## Design system (non-negotiable, keep everything consistent to this)
- **Color tokens only:** `--c50` (#F3F3F3 page bg) → `--c950` (#080808). Plus `--yellow:#FED259` (Bearings app accent; used inside app screens, not the case-study chrome).
- **Fonts:** Instrument Sans (`--sans`, everything) + Fraunces (`--serif`, display accents; blocked in the sandbox so it falls back to Georgia in Playwright but renders fine on Netlify).
- **Ink texture** via SVG filters: `#inkBig` (display/h3), `#inkSm` (labels/fine). Apply so nothing looks flat next to textured elements.
- **Motion:** 8fps stop-motion via `steps()` everywhere — no easing curves. Exits brisk, entrances settle, transition in place (never animate layout position).
- **Type scale:** `--fs-h2` clamp(26,2.5vw,36) headings, `--fs-h3`(21-32), `--fs-h4`(19-28), `--fs-prose`(~20) body, `--fs-small`, `--fs-label`(13). Labels are 13px/600 UPPERCASE, letter-spacing .1em. Body is 400/c700. Differentiate by **weight**, not near-miss sizes (32 next to 36 reads as a clash — avoid).
- **Spacing tiers:** 24px label→heading→body; ~55px between blocks (`clamp(40px,5.5vh,60px)`); chapter gap `.sec{padding-top:clamp(72px,11vh,140px)}`.
- Sharp corners on frames/cards; rounded only on pill buttons. Hairline borders `rgba(8,8,8,.14)`.
- **No em-dashes in written prose.** Preserve Jayden's own words; only smooth copy when invited.

# ANIMATION LESSONS (hard-won)

- The 8fps grid is math, not vibe: frames = round(duration × 8); steps(n,end); 125ms is the atomic beat — "text lands, screen answers one frame later" reads as intention.
- Scan for BOTH pure-ease and MIXED declarations: `transition:opacity .3s ease,transform .3s steps(4,end)` hides from naive scans that skip anything containing "steps(". The one legal ease: pure-opacity overlay fades (documented exception).
- Demo grammar beats GIF grammar: dwell (1.5s) → fingertip on the REAL control → instant cut → 3s rest on the resolved frame → loop. Uniform timing + no rest + loop seam = "bad GIF".
- Tap indicators must be computed, not guessed: new-yellow-between-frames finds selections, typed-text diff clusters find fields, on-frame color blobs find buttons. TRAP: the largest new-yellow is usually the NEXT screen's chrome; the four equal blobs at y14 are the progress tracker, not a field. No verifiable target → no dot. Wrong finger < no finger.
- Control-on-contact resolves the NN/g autoplay objection: motion demonstrates, text never moves, first tap kills auto permanently, ticks jump, offscreen pauses (IO threshold .25), reduced-motion rests on the resolved frame with tap-to-step intact. Ship the resolved frame in markup; cut to frame 0 UNDER the develop blur so nothing ever visibly jumps backward.
- Reveal-system testing: only trust real input (mouse.wheel). Scripted scrollTo/scrollBy against scroll-behavior:smooth retargets the animation and produces phantom "stuck" elements that no real user can trigger; it also self-heals on revisit.

# PERFORMANCE LESSONS

- CLS to literal 0: preload every woff2 (they're ~17KB each) + metrics-matched fallback faces. Compute overrides from @capsizecss/metrics (npm, allowed domain): size-adjust=(xWidthAvg/upm)/(fallback xWidthAvg/upm), ascent/descent-override=metric/(upm×sizeAdjust). Fraunces never loads in this sandbox (Google Fonts 403) — Georgia stands in; verify the real swap once on Netlify.
- Attribute shifts before fixing: PerformanceObserver layout-shift with sources[].node names the culprit in one run.
- SNAP TRAP: if a scroller's max scroll is smaller than one snap interval, proximity snap yanks every release back to 0 — feels like "can't scroll to the end". Fix: :last-child{scroll-snap-align:end} + drag inertia that keeps .dragging (snap off) during the glide and hands the landing to native snap.
- Lazy images inside horizontal scrollers don't load on vertical scroll; flip loading off when the container reveals (the dv observer already fires there).
- Delete dead weight on sight: the unused Fraunces request on bearings, the never-called headPeek machine. grep for callers before believing a function is alive.
- Audit reality, not source: render, then histogram computed font-sizes against the token set and measure actual gaps — margin COLLAPSE means the rendered gap is max(bottom,top), which is how 24/20 declarations read as 24, and how a missing margin-bottom read as 20.
